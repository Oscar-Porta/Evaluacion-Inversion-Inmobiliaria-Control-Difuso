3.Definicin de variables y universos de discurso

A continuacin definimos las variables que forman parte del sistema difuso:

Variables de entrada (antecedentes):

  • Precio por metro cuadrado: de 500 a 5000 USD
  • Rentabilidad esperada (% anual): de 0% a 20%
  • Ubicacin (calificacin subjetiva): de 0 a 10

Variable de salida (consecuente):

  • Conveniencia de inversin: de 0 a 100

Para cada variable definiremos su universo de discurso y las funciones de pertenencia correspondientes, usando funciones trapezoidales y sigmoidales segn las caractersticas proporcionadas en el enunciado.

In[8]:
# Definicin de los universos de discurso
universo_precio = np.arange(500, 5001, 1)         # Precio por m
universo_rentabilidad = np.arange(0, 20.1, 0.1)   # Rentabilidad %
universo_ubicacion = np.arange(0, 10.1, 0.1)      # Ubicacin
universo_conveniencia = np.arange(0, 101, 1)      # Conveniencia (salida)

# Creamos las variables difusas
precio = ctrl.Antecedent(universo_precio, 'precio')
rentabilidad = ctrl.Antecedent(universo_rentabilidad, 'rentabilidad')
ubicacion = ctrl.Antecedent(universo_ubicacion, 'ubicacion')
conveniencia = ctrl.Consequent(universo_conveniencia, 'conveniencia')

Funciones de pertenencia: Precio por metro cuadrado

Para la variable precio por metro cuadrado, definimos tres conjuntos difusos utilizando funciones de pertenencia trapezoidales:

  • Barato: desde 500 hasta aproximadamente 1500 USD/m
  • Medio: valores intermedios entre 1200 y 3000 USD/m
  • Caro: a partir de 2800 hasta 5000 USD/m

Estas funciones permiten que un mismo valor pueda pertenecer a ms de un conjunto con distinto grado, lo que permite razonamientos ms flexibles.

Funciones de pertenencia: Conveniencia de inversin

La variable conveniencia de inversin representa la salida del sistema, en una escala de 0 a 100. Usamos funciones de pertenencia sigmoidales para modelar dos categoras:

  • Baja: funcin sigmoidal decreciente con parmetros b=40, c=0.1
  • Alta: funcin sigmoidal creciente con parmetros b=60, c=0.1

Esto permite transiciones suaves entre baja y alta conveniencia, sin cortes bruscos.

# Funciones de pertenencia para la salida: conveniencia

conveniencia['baja'] = fuzz.sigmf(conveniencia.universe, b=40, c=-0.1)

4.Reglas difusas del sistema

A continuacin definimos el conjunto de reglas que forman la base del sistema de inferencia difuso. Estas reglas combinan los valores lingsticos de las variables de entrada y determinan si la conveniencia de inversin es alta o baja.

Reglas utilizadas:

  1. Si el precio es barato, la rentabilidad es alta y la ubicacin es buena, entonces la conveniencia es alta
  2. Si el precio es medio, la rentabilidad es alta y la ubicacin es buena, entonces la conveniencia es alta
  3. Si el precio es caro y la rentabilidad es baja, entonces la conveniencia es baja
  4. Si el precio es caro y la ubicacin es mala, entonces la conveniencia es baja
  5. Si la rentabilidad es alta y la ubicacin es media, entonces la conveniencia es alta
  6. Si la rentabilidad es baja y la ubicacin es mala, entonces la conveniencia es baja
In[19]:
# Definicin de las reglas difusas
regla_1 = ctrl.Rule(precio['barato'] & rentabilidad['alta'] & ubicacion['buena'], conveniencia['alta'], label='Regla 1')
regla_2 = ctrl.Rule(precio['medio']  & rentabilidad['alta'] & ubicacion['buena'], conveniencia['alta'], label='Regla 2')
regla_3 = ctrl.Rule(precio['caro']   & rentabilidad['baja'],                   conveniencia['baja'], label='Regla 3')
regla_4 = ctrl.Rule(precio['caro']   & ubicacion['mala'],                      conveniencia['baja'], label='Regla 4')
regla_5 = ctrl.Rule(rentabilidad['alta'] & ubicacion['media'],                conveniencia['alta'], label='Regla 5')
regla_6 = ctrl.Rule(rentabilidad['baja'] & ubicacion['mala'],                 conveniencia['baja'], label='Regla 6')

5.Evaluacin del sistema difuso

Una vez definidas las reglas, creamos el sistema completo utilizando ControlSystem y lo ejecutamos con ControlSystemSimulation. Evaluaremos el escenario propuesto en el enunciado, introduciendo los siguientes valores:

  • Precio: 2200 USD/m
  • Rentabilidad esperada: 11%
  • Ubicacin: 8

El sistema devolver un valor numrico entre 0 y 100 que representa el grado de conveniencia de invertir en ese inmueble.

In[20]:
# Crear el sistema de control difuso
sistema_control = ctrl.ControlSystem([
    regla_1, regla_2, regla_3, regla_4, regla_5, regla_6
])
simulador = ctrl.ControlSystemSimulation(sistema_control)

# Establecer los valores del escenario propuesto
simulador.input['precio'] = 2200
simulador.input['rentabilidad'] = 11
simulador.input['ubicacion'] = 8

# Ejecutar la inferencia difusa
simulador.compute()

# Mostrar el resultado
print(f"Conveniencia de inversin: {simulador.output['conveniencia']:.2f}")

# Visualizacin de la salida
conveniencia.view(sim=simulador)

Conveniencia de inversin: 74.29
